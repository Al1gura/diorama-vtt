## 一、产品五维度

所有功能从这五个维度推导。画面品质不是第六维度——它是所有维度的底线约束。

| # | 维度 | 一句话 | 受众 |
|---|------|--------|------|
| ① | **场景呈现** | 玩家能看到立体的、空间上可信的场景 | 玩家 |
| ② | **场景因果** | 场景变化是真实的——因果关系，不是假装 | 玩家 |
| ③ | **模组管理** | GM 管理的是一个完整冒险，不是一张张孤立的地图 | GM |
| ④ | **交互逻辑** | 跑团中软件不打断游戏——GM 一气呵成 | GM |
| ⑤ | **准备成本** | GM 备团的投入可控——不被工具拖累 | GM |

### 五维度独立性验证

- ①呈现 vs ②因果：静态呈现 vs 动态变化真实性。2D VTT（虚拟桌游平台）能做①但永远做不到②
- ②因果 vs ④交互：④是 GM（游戏主持人）能执行操作，②是操作后世界真的变了
- ③模组 vs ⑤成本：管理能力 vs 时间投入，不同维度
- ①呈现 vs ⑤成本：GM 想给玩家看的 vs 为这个要花多少时间——核心张力
- ④交互 vs ⑤成本：直觉操作 vs 操作耗时，不是同一件事

## 二、产品边界

**是什么：** 用 Godot（开源游戏引擎）引擎做的 GM（游戏主持人）桌面工具——开源 3D 俯视交互地图引擎。单 exe，GM 一人操作。

**第一场景：** 线下跑团，GM 电脑操作，投屏给全桌看。**线上功能初期不做，预留接口。**

**明确不做：** 联网同步、骰子系统、规则系统、模组市场、玩家端、账号。

**许可证：** MIT

## 三、编辑态 ↔ 运行态

一体化的单 exe 应用，一键无缝切换（类似 Unity 的 Editor ↔ Play 模式）。

- **编辑态**（备团）：地面材质绘制、物件拖放 + 属性标记、场景保存/加载
- **运行态**（跑团中）：Token（标记）拖拽、光源开关、特效触发、墙体破坏、投屏模式

切换时不丢失场景状态。

## 四、从五维度推导的功能清单

### 近期可做

| 功能 | 来自维度 | 说明 |
|------|---------|------|
| **正交相机 + 倾斜视角** | ① | 自由缩放/平移/旋转。默认倾斜角（非 90° 纯俯视）以保留立体感。90° 作为可选配置 |
| **网格地面** | ① | 可开关网格线。平面基底，高度差通过物件实现（拖放土丘/斜坡/凹陷物件） |
| **地面材质笔刷** | ①⑤ | 拖拽绘制材质/颜色。铺底色用笔刷，不做程序化混合和智能过渡 |
| **物件系统** | ①②③④⑤ | 物件仓库拖放。墙体（边缘单面/中间有厚度、可破坏）、地形物件（土丘/斜坡/岩石/台阶）、装饰物件（树/灌木/建筑/机关）、光源物件（火把/篝火/魔法光，可开关）、Token（角色/怪物，2D BillBoard（告示板渲染）默认 + 3D 模型可选） |
| **物件属性标记** | ②④ | 可破坏/可交互/光源参数/阵营等游戏数据 |
| **Token 拖拽** | ④ | 编辑态放 Token → 运行态可拖拽移动。吸附网格 |
| **光源 + 实时阴影** | ①② | 环境光 + 区域光源（颜色/范围/亮度可调）。实时阴影。直接光+阴影即可，GI（全局光照）反射后期加 |
| **LOS（视线遮挡）** | ①② | 墙体遮挡视线。破坏后 LOS 重算 |
| **墙体破坏** | ②④ | 运行态触发，Mesh（网格）碎裂 + 光照重新计算 + LOS 更新 |
| **特效触发** | ①② | 预设法术 VFX（视觉特效）（火球/闪电/治疗），运行态手动触发。粒子 + 光照变化 |
| **场景气氛设置** | ① | 环境光 + 体积雾（密度/颜色/范围）。Godot Environment（环境）节点已有 |
| **色块布局工具** | ⑤ | 新建场景时可用矩形/多边形简画房间布局，作为后续编辑参考 |
| **地图内层级切换** | ③ | 同一场景内的楼层跳转（不同 Z 轴层），前期做相机 Z 轴跳转 |
| **多场景管理 + 切换** | ③ | 一个模组包含多个场景文件，可切换 |
| **场景保存/加载** | ③ | 状态持久化（Token 位置、光源状态、墙体破坏状态） |
| **投屏模式** | ③④ | 一键全屏输出到第二屏。投屏无 UI（用户界面），仅 3D 场景 |
| **编辑↔运行切换** | ④ | 一键切换，工具栏变化 |

### 远期规划（规则系统 + 线上扩展）

| 功能 | 来自维度 | 说明 |
|------|---------|------|
| **移动范围计算** | ④ | 拖动 Token 时高亮可达范围。需要基础角色数据（速度值） |
| **战斗轮次** | ④ | 手动先攻列表 + 轮次管理。初期 GM 手填，不做自动规则计算 |
| **技能面板** | ④ | 预设法术/技能列表，点击触发效果。依赖规则系统数据 |
| **线上连接** | ③ | 预留的联网接口启用 |

## 五、开发优先级

| 优先级 | 功能 | 理由 |
|--------|------|------|
| **P0** | 正交相机 + 网格地面 + 光源 + 阴影 | 地基。先验证 Godot 渲染能不能看。含倾斜视角 |
| **P1** | 编辑↔运行切换 | 切换框架（运行态先空着），让 P1 后面有地方能切过去 |
| **P1** | 地面材质笔刷 | 编辑态基础 |
| **P1** | 物件系统 | 物件仓库+拖放+分类（墙体/地形/装饰/光源/Token） |
| **P1** | 物件属性标记 | 可破坏/可交互/光源参数/阵营 |
| **P1** | 场景保存/加载 | GM 搭完必须能存。依赖物件系统 |
| **P2** | Token 拖拽 | 编辑态放 Token → 切运行态可拖拽移动 |
| **P2** | LOS 视线遮挡 | 墙体遮挡视线 |
| **P2** | 墙体破坏 | 依赖 LOS + 墙体物件。Mesh 碎裂+光照重算+LOS 更新 |
| **P3** | 特效触发 | 预设法术 VFX（视觉特效），运行态手动触发 |
| **P3** | 场景气氛（雾+环境光） | 与光照和光源系统联动 |
| **P3** | 色块布局工具 | 提升准备效率，服务⑤ |
| **P4** | 多场景管理 + 切换 | 依赖场景保存/加载成熟 |
| **P4** | 地图内层级切换 | 依赖场景存在、Z 轴跳转 |
| **P4** | 投屏模式 | 依赖 P0-P3 场景有可看的内容 |
| **远期** | 移动范围计算 | 需要角色速度数据 |
| **远期** | 战斗轮次 | 初期 GM 手填先攻列表 |
| **远期** | 技能面板 | 依赖规则系统数据 |
| **远期** | 线上连接 | 联网接口启用 |
