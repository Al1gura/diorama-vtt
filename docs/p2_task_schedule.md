### 结构治理门禁

- 不以“文件变短”作为成功标准；必须减少共享可变状态，并让每次输入、选择、场景会话各有唯一所有者。
- 每阶段先写会失败的行为测试，再迁移实现；保留少量 `Viewport.push_input()` 端到端测试，新增控制器公开接口单元测试，逐步减少直接访问 `main.gd` 私有成员的 65 处测试耦合。
- 每阶段提交 Godot 4.7 源码对照卡，明确引擎行为、本地映射、不能照搬之处和自动/可见窗口验证。
- 不同时重写多个阶段；当前阶段完整回归、保存读回和 GM 关键手感通过后，才开始下一阶段。
- 结构治理期间不顺带实现 P2.3 光源、P2.4 战斗碰撞、P2.5 LOS 或 P2.6 墙体破坏，避免把行为新增和职责迁移混成一次修改。

## 目标边界

P2 的本质不是先做某个按钮，而是把“从资产栏位导入的东西，成为有类型、有属性、有运行态操作的对象”这层地基补上。

明确不做：自动骰子、完整自动规则引擎、自动伤害结算、完整职业技能规则、自动战斗轮次、联网玩家端。局部规则辅助不在禁区内，但涉及 CPR 时必须按 `docs/design.md` 的“CPR 规则资料查阅门禁”查证并标页码。

## 运行态按钮归属

P2.1 的操作面板只负责“选中对象后显示当前状态和可用入口”，不继续塞具体玩法按钮。后续按钮按对象和系统归属拆开实现，避免一个面板任务吞掉光源、战斗、LOS 和墙体破坏。

| 运行态按钮/入口 | 归属阶段 | 口径 |
|----------------|----------|------|
| Token 移动 | P2.2 已完成 | 继续直接拖动 Token，不新增重复“移动”按钮；运行态拖动是临时状态，回编辑态恢复底稿位置 |
| Token 射击线/瞄准线 | P2.4 | 只在运行态启用；做射击线/子弹线几何预览和 CombatBody 遮挡接口，不做命中、骰子、伤害 |
| Token 技能 | 后置 | 只可先做入口；完整技能面板依赖规则系统数据，不并入 P2.1 |
| 光源开关 | P2.3 | 运行态按钮切换 `LightProperties.is_on`，状态保存/读回 |
| 墙体破坏/修复 | P2.6 | 运行态按钮改变墙体状态，并同步 LOS/挡枪线阻挡语义，碎裂视觉后置 |
| 交互物体触发/启用禁用 | P2.3 前置小任务或 P2.3 同轮 | 可作为“运行态操作按钮框架”的首个轻量用例；只改 `InteractableProperties` 状态，不做规则系统 |

### P2.4 与 P2.5 的固定交接

- P2.4 交付 `OcclusionGeometry`：真实模型子树到物件根局部边界的唯一入口。
- P2.4 交付 `CombatBody` 与战斗线查询：只处理三维射击线几何遮挡。
- P2.5 直接读取 `OcclusionGeometry` 的局部边界，追加二维地面投影与可见区域；不读取 `CombatBody` 运行时形状。
- `blocks_shot` 与 `blocks_los` 分别控制两套输出。墙体破坏可同时关闭二者，但不能合并字段。

## 验收原则

- 涉及 CPR 规则时必须先查 `docs/cpr_reading/agent_index/`，验收记录必须带资料名与页码。
- 每个阶段都必须保存/加载验证，不只看当场运行。
- 运行态功能必须验证投屏窗口能看到正确结果，GM-only 信息不应泄漏到玩家画面。
- `PickProxy` 只负责点击拾取；战斗线用 CombatBody；LOS 用 LOSOccluder 或其阶段性替代体。
- 橙色选择 Gizmo 与移动阻挡盒采用等价的物体根局部边界逻辑，但各自计算、职责独立；不能把“看起来是同一个框”写成共用同一个运行对象或缓存。
- 技能、命中、伤害只允许做入口或几何预览，不在 P2 做规则计算。

## P2.2 完成条件

- Token 有 GM 可填写或导入的 CPR `MOVE`（移速）值。
- 按下并拖动 Token 时自动固定本次起点、查询路线并显示预算内路线，不增加可见的开始/结束按钮。
- 当前自由测距模式不得超过 `MOVE × 2` 米/码。以后增加网格移动模式时必须按每格 2 米/码、最多 `MOVE` 格并允许斜向移动，不能重新引入 1 米吸附假设。
- 超出范围时不显示圈外完整路线，只显示并提交预算内路线；无障碍直线移动终点为路线与范围圆的交点，绕障和高耗费地形可提前停在圈内。
- P2.2 每次拖动是一条移动命令；完整先攻/轮次系统尚未实现，是否允许下一条移动命令由 GM 决定，软件不冒充已经自动记账。
- CPR 的“跑”会消耗其他动作来再获得一次移动动作，属于紧邻后续项；基础 P2.2 不应默认把这第二次移动白送给 Token。
- 编辑态 Token 位置能保存/读回；运行态 Token 拖动不写回编辑态底稿、不标记场景未保存，回编辑态恢复进入运行态前的位置；投屏共享同一世界；GM 手动确认拖动手感。
- 新建或缺少 CPR 组件的 Token 默认 `MOVE=5`，即基础预算 10 米/码；已有存档中的明确 `MOVE` 值不自动覆盖。
- 按住 Token 开始拖动时显示以本次起点为圆心、当前规则预算为半径的基础距离圈；路线暂时无效时圈仍保留，松手/取消后消失。超距鼠标目标被截到预算内路线终点，真实绕障与地形耗费由路线成本决定。
- 短路线保持基础 6 米/秒，长路线自动提速并在 2 秒内走完；结束朝向采用最后一段水平路线方向，项目模型正面统一为本地 `+Z`。
- Token 寻路必须读取自身 `collision_radius/collision_height`：默认 Token 能通过 1.5 米净宽测试通道，半径 0.9 米的肥大 Token 必须使用另一张缓存导航图，不能借用默认体型路线。
- 非地形阻挡物使用重建期缓存的局部边界盒并保留旋转/缩放；地形、坡面、台阶使用三角网格碰撞。导航精度固定为水平 0.1 米、垂直 0.25 米，退出运行态或重建时释放所有缓存导航资源。

规则依据：`docs/cpr_reading/agent_index/agent_chunks.jsonl` 中《赛博朋克 RED 核心规则书》PDF p.144（书内 p.126）、PDF p.145（书内 p.127）和 PDF p.187（书内 p.169），对应块均为 `needs_pdf_check=false`。
