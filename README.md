# Diorama VTT

> 一个可交互的 3D 俯视虚拟跑团桌 / An Interactive 3D Top-Down Virtual Tabletop

---

## 核心理念 / Core Concept

**「一张活的立体地形桌」——A Living 3D Diorama Table**

传统的 VTT 要么是平面 2D 地图，要么是博德之门式的 3D 沉浸视角。Diorama VTT 选择了一条中间路线：

- **俯视（Top-down Orthographic）** — 像看实体桌面一样看地图，适合线下投屏 + 实物 Token
- **3D 地形** — 地形、建筑、物件是真实 3D 体块，有体积、有厚度、可交互
- **实时光照** — 火炬照亮黑暗、法术爆炸瞬间照亮周围、墙挡住视线
- **地图是活的** — 墙可以被轰开、火把可以点燃、地形可以被改变

**这不是另一个 Foundry VTT 的替代品。这是一套「3D 交互地图引擎」，规则层留给社区。**

---

## 为什么做这个？/ Why This Exists

| 现有方案 | 缺少什么 |
|---------|---------|
| **Foundry VTT** | 地图是静态 2D 图，墙是线段遮罩，不是真实几何体 |
| **TaleSpire** | 不支持 2D Token，不支持俯视视角，每人需要付费 |
| **Dungeon Alchemist** | 地图生成器，不是 VTT，跑团时地图是死的 |
| **Roll20 / Owlbear** | 平面 2D，无 3D 交互 |

Diorama VTT 填补的空白：**一个开源的、可交互的 3D 俯视地图引擎，让 GM 和玩家拥有真正「活的」跑团桌面。**

---

## 核心功能 / Core Features

### ✅ MVP 范围（预计 3-6 个月）

| 功能 | 说明 |
|------|------|
| **3D 俯视地图展示** | Godot 引擎，正交相机，自由缩放/平移 |
| **3D 地形搭建** | GridMap 拼搭系统 + CSG 几何体，无需外部资产即可建 dungeon |
| **动态光照系统** | 环境光源 + 可拖放火炬/篝火 + 实时阴影 + LOS 视线遮挡 |
| **可交互地形** | 墙可标记为可破坏 → 摧毁后 LOS 和光照实时更新 |
| **法术特效光照** | 火球/闪电/治疗等爆发式光照，几秒消退或持续 |
| **2D Token 系统** | Sprite3D BillBoard，PNG 立绘直接作为角色站在 3D 地形上 |
| **实物 Token 模式** | 显示位置标记 / 高亮格，配合线下实物棋子使用 |
| **骰子系统** | 内置骰子指令，也支持 GM 手动输入结果（兼容实物骰子） |
| **投屏模式** | 全屏 + 无 UI 输出，HDMI 投电视/投影仪 |
| **规则 API (GDScript)** | 暴露核心接口，让社区编写自定义规则脚本 |

### ❌ 明确不作（初期）

- 规则引擎 / 自动化（D&D 5e / COC / PF2e 等）— 交给社区通过 API 实现
- 多人网络同步 — 先做本地单机
- 模组系统 / 插件市场
- 地图编辑器精细度（参考 Minecraft 水平的简单拼搭）

## 架构 / Architecture

```
┌─────────────────────────────────────────────────────┐
│                    GM Client                         │
│  ┌───────────────────────────────────────────────┐  │
│  │                Presentation Layer              │  │
│  │  3D Scene (Camera/Lighting/Terrain/Tokens)    │  │
│  └──────────────────────┬────────────────────────┘  │
│  ┌──────────────────────┴────────────────────────┐  │
│  │              Interaction Layer                 │  │
│  │  Terrain Modify / Token Move / Light Toggle   │  │
│  └──────────────────────┬────────────────────────┘  │
│  ┌──────────────────────┴────────────────────────┐  │
│  │               Rule API Layer                   │  │
│  │  action(target, op, params) → effect/resolve  │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

**核心 API 抽象：**

```
操作 (Action)  →  目标/范围 (Target)  →  检定 (Check)*  →  结果 (Result)
```

- Check 是可选的 — 无检定 = 必然成功
- Result 可能是伤害、光照特效、地形改变、状态变更，或其任意组合
- API 不绑定任何特定规则系统，社区脚本决定怎么用

\*「检定」对软件来说只是一个随机数源。可以是骰子、Jenga 塔、抽牌、GM 拍脑袋决定——引擎只负责输入和输出。

## 为什么是 Godot？

| 维度 | 理由 |
|------|------|
| **3D 渲染** | 原生 Vulkan/Metal，性能远好于浏览器套壳 |
| **正交相机** | Camera3D 原生支持，俯视视角一行代码搞定 |
| **光照系统** | 完整的 3D 实时光照 + 阴影管线 |
| **GridMap** | 内建体素拼搭系统，适合快速搭建 3D 地形 |
| **CSG** | 无需外部 3D 模型就能用几何体搭 dungeon |
| **2D Sprite 在 3D 空间** | Sprite3D / BillBoard 原生支持 2D 立绘在 3D 场景中渲染 |
| **成本** | MIT 协议，完全免费 |
| **脚本语言** | GDScript 轻量易学，规则脚本门槛低 |
| **导出** | 可打包 exe（线下投屏）或 Web（网团分享） |

## 理念 / Philosophy

> **Diorama VTT 不信奉「做大而全」。**
> 
> Foundry 已经是大而全的终点。我们不做另一个 Foundry。
>
> 我们做的是一个**开源的、可交互的 3D 地图引擎**，核心卖点是「活的立体地形桌」。规则自动化交给社区和 API。
